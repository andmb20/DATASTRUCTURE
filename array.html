Array

"Array é uma coleção de elementos ordenada, acessíveis por índice, armazenados em memória contígua (que estão um ao lado do outro)."

Um array é uma estrutura de dados linear que armazena elementos do mesmo tipo (homogêneos) ou de tipos diferentes (heterogêneos, dependendo da linguagem) em posições contíguas de memória, acessíveis por um índice numérico.

    Características Principais

  1. Coleção Ordenada

    2. Os elementos têm uma ordem definida (índices começando em 0 ou 1, dependendo da linguagem).

  2. Acesso Direto por Índice

    3. Permite acesso rápido a qualquer elemento em O(1) (tempo constante).

    Ex.: array[3] retorna o 4º elemento.

  3. Tamanho Fixo ou Dinâmico

    4. Estático: Tamanho definido na criação (ex.: em C, int vetor[10];).

    Dinâmico: Cresce/shrink conforme necessário (ex.: em JavaScript/Python).

  4. Uso Eficiente de Memória

    5. Elementos são armazenados em blocos contíguos, facilitando iteração e cache da CPU.

  5. Operações Básicas

    6. Inserção, remoção, busca, ordenação (cada uma com sua complexidade, como vimos antes).

Exemplos em Diferentes Linguagens

// JavaScript (Array dinâmico e heterogêneo)
let arrayJS = [1, "texto", true];
console.log(arrayJS[1]); // "texto"

# Python (List = Array dinâmico)
lista = [10, 20, 30]
lista.append(40) # Adiciona no final

// C (Array estático e homogêneo)
int vetor[5] = {1, 2, 3, 4, 5};
printf("%d", vetor[2]); // 3

Complexidade das Operações

Operação Complexidade Exemplo em JavaScript
Acesso por índice O(1) array[2]
Inserir no final O(1)* array.push("item")
Inserir no início O(n) array.unshift("item")
Remover do final O(1) array.pop()
Remover do início O(n) array.shift()
Buscar por valor O(n) array.indexOf("item")
Ordenar O(n log n) array.sort()

*Obs: Em linguagens com arrays dinâmicos (JavaScript/Python), push é geralmente O(1), mas pode ser O(n) se o array precisar ser realocado na memória.


    Quando Usar Arrays?

    Use arrays quando:

  • Você precisa de acesso rápido por índice (ex.: lista[3]).
  • A ordem dos elementos é importante.
  • Exemplos clássicos:
    • Lista de alunos em uma turma.
    • Pixels de uma imagem (matriz = array 2D).
    • Buffer de dados sequenciais

    Analogia Prática

    Pense em um array como uma prateleira de livros:

  • Cada livro tem um número de posição fixo (índice).
  • Você pode pegar o livro diretamente se souber sua posição.
  • Adicionar/remover livros no meio exige reorganização (O(n)).

    Quando NÃO Usar Arrays?

  • Quando precisar de inserções/remoções frequentes no meio (use listas ligadas).
  • Quando a ordem não importa e quer elementos únicos (use sets).
  • Quando precisa de mapeamento chave-valor (use dicionários).
